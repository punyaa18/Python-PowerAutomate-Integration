# Contributing to Ollama Integration Project

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker
- Describe the issue clearly with steps to reproduce
- Include your OS, Python version, and Ollama version
- Provide error messages and logs if applicable

### Submitting Changes

1. **Fork the repository**
   ```bash
   git fork https://github.com/yourusername/ollama-power-automate-vscode
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow Python PEP 8 style guidelines
   - Add comments to complex code
   - Update documentation as needed

4. **Test your changes**
   - Ensure all examples run without errors
   - Test with different Ollama models
   - Verify Power Automate integrations work

5. **Commit with clear messages**
   ```bash
   git commit -m "Add: Brief description of changes"
   ```

6. **Push and create a Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```

## Code Style

- Use 4 spaces for indentation
- Follow PEP 8 naming conventions
- Add docstrings to all functions
- Keep functions focused and small

## Adding Examples

When adding new examples:

1. Create a new `.py` file in the `examples/` directory
2. Include a module docstring explaining the purpose
3. Add inline comments for complex logic
4. Update the README.md with the new example

## Questions?

Feel free to open an issue for questions or clarifications.
